# 要求内容

## 概要

未ログイン状態の誰でも `/register` から親（PARENT ロール）アカウントを自己登録できる機能を実装する。登録時に新しい家族（familyId）が自動生成される。

## 背景

現在は DataInitializer のシードデータにより親アカウントが作成されるだけであり、新しい家族が実際にアプリを使い始める手段がない。自己登録フォームを設けることで、セットアップ不要で家族ごとに独立した利用が可能になる。

## 実装対象の機能

### 1. 親アカウント登録フォーム（GET /register）

- ログイン不要でアクセスできる
- 以下のフィールドを持つフォームを表示する
  - ユーザーID（必須、3〜20文字、英数字・アンダースコアのみ）
  - パスワード（必須、8文字以上）
  - パスワード確認（必須、パスワードと一致すること）
  - メールアドレス（必須）
  - 表示名（任意、50文字以内）

### 2. 親アカウント登録処理（POST /register）

- 入力値のバリデーション（Bean Validation）
- ユーザーID の重複確認
- メールアドレスの重複確認
- パスワードの一致確認
- BCrypt でパスワードをハッシュ化して保存
- role = PARENT、familyId = 保存後の自分自身の ID をセット
- 登録成功後は /login?registered=true にリダイレクト

### 3. ログインページからの導線

- ログインページに「アカウントを作成する」リンクを追加する

## 受け入れ条件

- [ ] 未ログイン状態で `/register` にアクセスできること
- [ ] 必須項目が空の場合バリデーションエラーが表示されること
- [ ] ユーザーID・メールアドレスが重複する場合エラーが表示されること
- [ ] パスワードと確認が一致しない場合エラーが表示されること
- [ ] 登録後に PARENT ロールのユーザーが作成されること
- [ ] 登録後に familyId が自身の ID と等しいこと
- [ ] 登録後は /login?registered=true にリダイレクトされること
- [ ] ログインページから `/register` へのリンクが表示されること

## スコープ外

- ログイン済みユーザーが親アカウントを追加登録する機能
- メール確認（email verification）
- 招待コードによる家族グループへの参加
