# 要求内容

## 概要

ユーザーID（または メールアドレス）とパスワードでログインし、親ロール・子どもロールに応じて機能を切り替えるログイン機能を実装する。

## 背景

お手伝いポイント管理アプリはブラウザ上で動作するWebアプリ。ログイン機能がないとすべてのデータが誰でも参照・操作でき、セキュリティ上成立しない。また親と子どもで使える機能が異なるため、ロールベースのアクセス制御が必須。

## 実装対象の機能

### 1. ログイン
- ユーザーID または メールアドレス + パスワードで認証
- 認証成功後はロールに応じて遷移先を分岐（PARENT→/approvals、CHILD→/chores）
- 認証失敗時はログイン画面にエラーメッセージを表示

### 2. ログアウト
- ログアウト操作でセッションを破棄し、ログイン画面へリダイレクト

### 3. ロールベースアクセス制御
- `/chores/**`, `/requests/**` → ROLE_CHILD のみアクセス可
- `/approvals/**`, `/history` → ROLE_PARENT のみアクセス可
- 未認証は全URLをログイン画面へリダイレクト

### 4. 初期データ投入
- 開発・動作確認用に親ユーザーと子どもユーザーのサンプルデータをDBに投入する仕組みを用意

## 受け入れ条件

### ログイン
- [ ] ユーザーID（userId フィールド）でログインできる
- [ ] メールアドレス（email フィールド）でログインできる
- [ ] 認証失敗時に「ユーザーIDまたはパスワードが違います」と表示される
- [ ] 認証成功後、PARENT は `/approvals` へ、CHILD は `/chores` へリダイレクトされる
- [ ] セッションはブラウザを閉じるまで維持される

### ロールアクセス制御
- [ ] 未ログイン状態でURLに直接アクセスすると `/login` にリダイレクトされる
- [ ] CHILD ロールで `/approvals` にアクセスすると 403 または ログイン画面にリダイレクト
- [ ] PARENT ロールで `/chores` にアクセスすると 403 または リダイレクト

### セキュリティ
- [ ] パスワードは BCrypt でハッシュ化されて保存される
- [ ] CSRF トークンがフォームに含まれる

## スコープ外

- パスワードリセット機能
- 新規ユーザー登録画面（初期データは DataInitializer で投入）
- メール通知
- 2段階認証
