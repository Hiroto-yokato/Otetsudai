# 要求内容

## 概要

子ども（CHILD ロール）が日付と内容を入力してお手伝いを記録し、一覧で確認・削除できる機能を実装する。

## 背景

ログイン機能が完成し、子どもがアプリにアクセスできるようになった。次のステップとして、お手伝いの実績を登録する中核機能が必要。`functional-design.md` の ChoreController / ChoreService / ChoreRepository に相当する実装を行う。

## 実装対象の機能

### 1. お手伝い一覧表示（GET /chores）

- ログイン中の子どものお手伝い一覧を表示する
- 日付降順で表示する
- ステータス（未申請・申請中・承認済み・却下）をバッジで表示する
- 「登録する」ボタンから登録フォームへ遷移する
- 登録完了後に `?registered=true` パラメータで完了アニメーションを表示する

### 2. お手伝い登録（GET /chores/new, POST /chores）

- 日付と内容（1〜100文字）を入力して登録する
- 未来日は入力不可（Bean Validation: `@PastOrPresent`）
- バリデーションエラー時はフォームに戻りエラーを表示する
- 登録成功時は一覧ページにリダイレクトし完了アニメーションを表示する

### 3. お手伝い削除（POST /chores/{id}/delete）

- UNPAID（未申請）状態のお手伝いのみ削除できる
- 他の状態（PENDING/APPROVED/REJECTED）では削除ボタンを表示しない
- 他ユーザーのお手伝いは削除不可（Service 層でチェック）

## 受け入れ条件

### お手伝い一覧表示
- [ ] CHILD ロールでログインすると `/chores` にアクセスできること
- [ ] 自分のお手伝いが日付降順で一覧表示されること
- [ ] 各お手伝いにステータスバッジが表示されること
- [ ] `?registered=true` のとき完了アニメーションが表示されること

### お手伝い登録
- [ ] GET /chores/new にアクセスすると登録フォームが表示されること
- [ ] 日付・内容を入力して送信するとお手伝いが UNPAID で保存されること
- [ ] 内容が空の場合はエラーメッセージが表示されること
- [ ] 内容が 101 文字以上の場合はエラーメッセージが表示されること
- [ ] 未来日を選択した場合はエラーメッセージが表示されること
- [ ] 登録成功後は `/chores?registered=true` にリダイレクトされること

### お手伝い削除
- [ ] UNPAID のお手伝いに削除ボタンが表示されること
- [ ] PENDING/APPROVED/REJECTED では削除ボタンが表示されないこと
- [ ] 削除後は一覧からそのお手伝いが消えること
- [ ] 他ユーザーのお手伝いを削除しようとすると例外が投げられること

## スコープ外

以下はこのフェーズでは実装しません:

- お手伝いの編集機能
- お小遣い申請機能（次フェーズ）
- ページネーション（件数が少ないため MVP では不要）
- REJECTED 後の UNPAID 戻し処理（申請機能と同時に実装）
