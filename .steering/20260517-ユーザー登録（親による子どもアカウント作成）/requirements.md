# 要求内容

## 概要

親（PARENT ロール）が子どものアカウントを作成する機能を実装する。登録フォームからユーザーID・パスワード・メールアドレス（任意）・表示名（任意）を入力して CHILD ロールのユーザーを作成する。

## 背景

現在はアプリ起動時の DataInitializer でシードデータとして親・子どもアカウントが作成されるだけであり、家族が実際に使う際には自分たちのアカウントを作れない。親が子どものアカウントを管理できる仕組みが必要。

## 実装対象の機能

### 1. 子どもアカウント一覧（GET /users）
- 親が自分の family_id に属する CHILD ロールのユーザー一覧を確認できる
- 各ユーザーの userId・表示名・登録日を表示する
- 「子どもを追加する」ボタンを表示する

### 2. 子どもアカウント登録フォーム（GET /users/new）
- 以下のフィールドを持つフォームを表示する
  - ユーザーID（必須、3〜20文字、英数字・アンダースコアのみ）
  - パスワード（必須、8文字以上）
  - パスワード確認（必須、パスワードと一致すること）
  - メールアドレス（任意）
  - 表示名（任意、50文字以内）

### 3. 子どもアカウント登録処理（POST /users）
- 入力値のバリデーション（Bean Validation）
- ユーザーID の重複確認（既存ユーザーと被らないこと）
- メールアドレスの重複確認（設定する場合）
- パスワードの一致確認
- BCrypt でパスワードをハッシュ化して保存
- role = CHILD、familyId = 親と同じ値をセット
- 登録成功後は /users?registered=true にリダイレクト

## 必要なスキーマ変更

User エンティティに `displayName` フィールドを追加する（nullable、最大50文字）。`hibernate.ddl-auto=update` により自動反映される。

## 受け入れ条件

### 登録フォーム
- [ ] PARENT ロールでログインすると `/users/new` にアクセスできること
- [ ] 必須項目が空の場合バリデーションエラーが表示されること
- [ ] ユーザーIDが重複する場合エラーメッセージが表示されること
- [ ] パスワードと確認が一致しない場合エラーが表示されること

### 登録処理
- [ ] 登録後に CHILD ロールのユーザーが作成されること
- [ ] パスワードが BCrypt でハッシュ化されること
- [ ] familyId が親と同じ値になること
- [ ] 登録後は /users?registered=true にリダイレクトされること

### 一覧
- [ ] `/users` で自分の家族の子どもアカウント一覧が表示されること

## スコープ外

- アカウント削除・編集
- パスワードリセット
- 親アカウント自身の登録（初回セットアップはシードデータで対応）
